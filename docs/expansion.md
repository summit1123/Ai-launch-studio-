# 확장 로드맵 (MVP 이후)

## 1. 확장 원칙
MVP는 홍보 문제를 먼저 해결하고, 사용자 다음 문제(운영/재고/매출)로 자연 확장한다.

## 2. 단계별 확장

### 단계 A: 홍보 최적화
- 캠페인 성과 입력
- 성과 기반 주간 플랜 자동 업데이트
- 채널별 재배분 추천

### 단계 B: 커머스 운영
- 재고 위험 알림
- 할인/가격 시뮬레이션
- 퍼널 병목 진단

### 단계 C: 리테일 오케스트레이션
- 홍보/재고/마진 통합 최적화
- 승인 기반 운영 대시보드
- 정책 기반 자동 제안

## 3. 에이전트 확장
- 단계 A: 성과 분석 에이전트
- 단계 B: 재고/매출 에이전트
- 단계 C: 통합 리테일 오케스트레이터

## 4. 확장 전제 데이터
- 광고 성과 데이터
- 주문/반품 이벤트
- 재고/원가/마진 데이터

## 5. 준비 체크리스트
- API 버전 분리
- 데이터 모델 확장 가능성 확보
- 팀 권한/승인 구조 설계
